Getting Started

Installation

The main CHAI distribution file unzips to a set of directories that is ready for compilation and linking. All relevant header files are in the main 'include' directory. To link against the CHAI libraries, use the .lib files in the lib/msvc6 or lib/bbcp6 directories.

If you want to modify and/or rebuild CHAI, project files for MSVC6 are located in the 'msvc6' directory, and project files for Borland C++ 6 are located in the 'bbcp6' directory. Both projects build straight to the appropriate lib directory.

To run CHAI applications, you should include the main 'bin' directory in your system path. Relevant dll's for interfacing with haptic devices are located in this directory.

If you are planning to use the CHAI interface to the ServoToGo I/O board, you'll also need to set up the dlportio library, which we've included in the main bin directory. You can just copy DLPORTIO.sys into your system32/drivers directory, or you can use the installer (port95nt.exe) provided by the manufacturer (DriverLinx), which we've also included in the 'external' directory. If Borland users have any link-time or run-time trouble with the DLPORTIO libraries provided in the DriverLinx installer, there is a free wrapper available.

Getting started with CHAI development

Although full doxygen documentation can be downloaded from the docs page, the best way to get started with CHAI is to take a look at the examples that are provided in the main distribution. Examples are provided for MSVC 6.0 and Borland C++ 6.00. Each example includes a README with helpful hints about where the "highlights" in in the code are.

The first two examples demonstrate the core features of CHAI, with the second example building on the first:

basic_shapes renders a single mesh, and allows the user to interact with it haptically using a proxy.

object_loader allows the user to load mesh files (in .3ds and .obj format) and to manipulate various haptic and graphic rendering options (including stereo display).

The mass_springs example demonstrates a simple dynamic environment, in which a series of masses are connected by springs, with the haptic device controlling one of the masses. This demonstrates the use of custom graphic and haptic rendering loops that override the default object behavior.

The board_tester, math, and clock examples are specific demonstrations of the i/o board, matrix math, and precision timer (respectively) features of CHAI. They demonstrate these features in isolation and do not create a virtual world.
